1. JavaScript BigInt

BigInt is a built-in JavaScript data type that can represent integers with arbitrary precision. It is useful when dealing with numbers that are too large to be represented by the regular Number type, which is limited to 64-bit floating-point values.

1.1 When to Use BigInt

BigInt is helpful when working with values that exceed the range of the Number type, such as large integers for cryptography, finance, or mathematical computations. While the Number type can represent integers up to 253 - 1, BigInt can represent much larger values.

1.2 Creating a BigInt

A BigInt can be created by appending the letter n to the end of an integer value or by using the BigInt() constructor.

let bigIntValue = 1234567890123456789012345678901234567890n;
let anotherBigInt = BigInt("1234567890123456789012345678901234567890");

1.3 Arithmetic with BigInt

BigInt supports most arithmetic operations just like regular numbers, including addition, subtraction, multiplication, and division. However, BigInts cannot be mixed with regular numbers in expressions. You must explicitly convert numbers to BigInts, or vice versa.

let bigInt1 = 1234567890123456789012345678901234567890n;
let bigInt2 = 9876543210987654321098765432109876543210n;
let sum = bigInt1 + bigInt2;
console.log(sum); // Outputs: 11111111101111111110111111111011111111100n

1.4 Limitations of BigInt

  • BigInt cannot be used with methods that expect a regular Number type, such as Math methods.
  • BigInt values cannot be used with the +"" operator or any methods designed for string manipulation that rely on the Number type.

2. JavaScript Number Methods

JavaScript provides several built-in methods that can be used with numbers for various purposes, such as formatting, rounding, parsing, and performing mathematical operations. Here are some of the most commonly used number methods:

2.1 toFixed()

The toFixed() method formats a number to a specified number of decimal places and returns the result as a string.

let num = 3.14159;
console.log(num.toFixed(2)); // Outputs: "3.14"

2.2 toPrecision()

The toPrecision() method formats a number to a specified length, which includes the total number of significant digits.

let num = 3.14159;
console.log(num.toPrecision(4)); // Outputs: "3.142"

2.3 toString()

The toString() method converts a number to its string representation.

let num = 123;
console.log(num.toString()); // Outputs: "123"

2.4 valueOf()

The valueOf() method returns the primitive value of a number object. This is often used when a number is created as an object.

let numObj = new Number(100);
console.log(numObj.valueOf()); // Outputs: 100

2.5 Math Methods

JavaScript provides the Math object, which includes several mathematical methods that can be used with numbers, such as:

  • Math.round() - Rounds a number to the nearest integer.
  • Math.floor() - Rounds a number down to the nearest integer.
  • Math.ceil() - Rounds a number up to the nearest integer.
  • Math.max() - Returns the largest of zero or more numbers.
  • Math.min() - Returns the smallest of zero or more numbers.
let num = 4.7;
console.log(Math.round(num)); // Outputs: 5
console.log(Math.floor(num)); // Outputs: 4
console.log(Math.ceil(num)); // Outputs: 5
console.log(Math.max(1, 5, 3)); // Outputs: 5

3. JavaScript Number Properties

JavaScript provides several built-in properties related to numbers that can be used to work with the Number data type:

3.1 Number.MAX_VALUE

The Number.MAX_VALUE property represents the largest positive number that can be represented in JavaScript.

console.log(Number.MAX_VALUE); // Outputs: 1.7976931348623157e+308

3.2 Number.MIN_VALUE

The Number.MIN_VALUE property represents the smallest positive number greater than zero that can be represented in JavaScript.

console.log(Number.MIN_VALUE); // Outputs: 5e-324

3.3 Number.POSITIVE_INFINITY

The Number.POSITIVE_INFINITY property represents positive infinity, which is a value greater than any finite number.

console.log(Number.POSITIVE_INFINITY); // Outputs: Infinity

3.4 Number.NEGATIVE_INFINITY

The Number.NEGATIVE_INFINITY property represents negative infinity, which is a value smaller than any finite number.

console.log(Number.NEGATIVE_INFINITY); // Outputs: -Infinity

3.5 Number.NaN

The Number.NaN property represents "Not-a-Number", a special value that indicates an invalid number operation (e.g., dividing 0 by 0).

console.log(Number.NaN); // Outputs: NaN

3.6 Number.EPSILON

The Number.EPSILON property represents the smallest difference between two representable numbers in JavaScript. It is used for precision checking when comparing numbers with floating-point values.

console.log(Number.EPSILON); // Outputs: 2.220446049250313e-16
