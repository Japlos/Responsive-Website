1. HTML JavaScript

JavaScript is a powerful scripting language used to create interactive effects within web browsers. It is commonly used to handle events, validate forms, manipulate DOM elements, and create dynamic content on web pages.

Basic Structure:

<script>
    // JavaScript code goes here
</script>

External JavaScript File:

To keep JavaScript code separate from HTML, you can link to an external JavaScript file using the <script> tag with the src attribute.

<script src="path/to/your/script.js"></script>

Example:

<script>
    alert("Hello, World!");
</script>

Why Use JavaScript?

  • It allows you to create interactive elements, such as sliders, forms, and buttons that respond to user actions.
  • It enables dynamic changes to the page content without reloading the page.
  • It integrates with other web technologies (e.g., HTML and CSS) to provide a rich user experience.

2. File Paths in HTML

File paths in HTML are used to specify the location of external resources like images, stylesheets, and JavaScript files. There are two types of file paths: relative and absolute.

Relative File Paths:

A relative path specifies the location of a file relative to the current document. It is useful when working with local files in a website directory structure.

<img src="images/photo.jpg">

Absolute File Paths:

An absolute path specifies the full URL to the file, starting from the root of the web or file system. This is commonly used for linking to external websites or resources.

<a href="https://www.example.com">Visit Example</a>

Example of Both:

<link rel="stylesheet" href="styles/main.css">
<script src="https://cdnjs.cloudflare.com/ajax/libs/jquery/3.6.0/jquery.min.js"></script>

3. HTML Head

The <head> element contains meta-information about the HTML document, such as the title, character set, linked stylesheets, and scripts that are needed before the page content is displayed.

Basic Structure:

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
    <link rel="stylesheet" href="styles.css">
    <script src="script.js"></script>
</head>

Common Elements Inside the <head> Tag:

  • <meta>: Provides metadata such as the character set and viewport settings.
  • <title>: Sets the title of the page (shown in the browser tab).
  • <link>: Links to external resources, such as stylesheets.
  • <script>: Links to external JavaScript files or contains inline JavaScript.

4. HTML Layout

Layout in HTML refers to the arrangement of elements on a page. You can structure a page using block-level and inline elements, as well as CSS for styling and positioning.

Common Layout Elements:

  • <header>: Defines the header section of the page.
  • <nav>: Defines a navigation section.
  • <main>: Specifies the main content of the page.
  • <footer>: Defines the footer section of the page.
  • <section>: Represents a thematic grouping of content.
  • <article>: Represents a self-contained piece of content.

Example Layout:

<header>
    <h1>Page Header</h1>
</header>

<nav>
    <ul>
        <li><a href="#home">Home</a></li>
        <li><a href="#about">About</a></li>
    </ul>
</nav>

<main>
    <section>Content section 1</section>
    <section>Content section 2</section>
</main>

<footer>
    <p>Footer Content</p>
</footer>

5. Responsive Web Design

Responsive web design ensures that web pages render well on a variety of devices and screen sizes. It uses techniques like fluid grids, flexible images, and CSS media queries to adapt the layout.

Key Concepts:

  • Fluid Layouts: Use relative units (like percentages) instead of fixed-width units (like pixels) to allow elements to resize based on screen size.
  • CSS Media Queries: Apply different CSS rules depending on the screen size or device.
  • Flexible Images: Images that resize within their container to fit different screen sizes.

Example of Media Query:

@media screen and (max-width: 768px) {
    body {
        font-size: 14px;
    }
}

6. HTML Semantics

Semantic HTML refers to the use of HTML tags that convey meaning about the content they contain. Using semantic elements improves accessibility, SEO, and maintainability of a web page.

Common Semantic Tags:

  • <header>: Represents the introductory content or navigation of a page.
  • <footer>: Represents the footer section, often containing copyright information or links.
  • <article>: Defines a self-contained piece of content that can be reused.
  • <section>: Represents a section of content within a document.
  • <aside>: Represents content that is tangentially related to the main content.

Benefits of Semantic HTML:

  • Improves SEO by making it clear to search engines what the content is about.
  • Enhances accessibility for users with screen readers.
  • Makes the code more readable and maintainable.

7. HTML Entities

HTML entities are used to represent special characters that cannot be typed directly into HTML code, such as symbols, non-Latin characters, or reserved characters like < and >.

Common HTML Entities:

  • & - Ampersand
  • < - Less-than
  • > - Greater-than
  • " - Double quotation mark
  • © - Copyright symbol

Example of Using Entities:

<p>Copyright © 2024</p>

8. HTML Style Guide

Following a style guide helps maintain consistency, readability, and maintainability of HTML code across a project. A good style guide ensures that everyone on the team writes HTML in a uniform way.

Common Style Guide Practices:

  • Use lowercase for all HTML tags and attributes.
  • Use consistent indentation (typically 2 or 4 spaces) for nested elements.
  • Close all tags, including self-closing tags like <img> and <br>.
  • Use semantic tags whenever possible for clarity and accessibility.
  • Separate external JavaScript and CSS files from HTML for maintainability.
